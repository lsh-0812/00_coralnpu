# 调试模块（Debug Module）

## 概述

调试模块实现了 RISC-V 调试规范（RISC-V Debug Specification）的一个子集，为 CoralNPU 核提供外部调试支持。它允许调试器停机（halt）、恢复（resume）和单步（single-step）核，以及读写核的寄存器和内存。

## 架构

调试模块是一个独立模块，通过专用的调试接口连接到核。它通过一个简单的请求/响应协议与外部调试器通信。

## 命令协议（Command Protocol）

外部调试器通过 AXI 接口读写一组内存映射的控制与状态寄存器（CSR）来与调试模块通信。这些 CSR 为调试模块的内部寄存器提供了一条通信通道。

### 写操作

要写入调试模块的某个内部寄存器（例如向地址 `0x10` 处的 `dmcontrol` 写入 `0x1`）：

1.  **轮询就绪状态：** 读取地址 `0x31014` 处的 `status` CSR，等待 bit 0 变为 `1`。
2.  **设置地址：** 将目标内部寄存器地址（`0x10`）写入地址 `0x31000` 处的 `req_addr` CSR。
3.  **设置数据：** 将数据（`0x1`）写入地址 `0x31004` 处的 `req_data` CSR。
4.  **发起写入：** 将 `WRITE` 操作码（`2`）写入地址 `0x31008` 处的 `req_op` CSR。
5.  **轮询响应：** 读取地址 `0x31014` 处的 `status` CSR，等待 bit 1 变为 `1`。
6.  **检查状态：** 读取地址 `0x31010` 处的 `rsp_op` CSR，确认操作成功。
7.  **应答响应：** 写入地址 `0x31014` 处的 `status` CSR 以清除该响应。

### 读操作

要读取调试模块的某个内部寄存器（例如从地址 `0x11` 处的 `dmstatus` 读取）：

1.  **轮询就绪状态：** 读取地址 `0x31014` 处的 `status` CSR，等待 bit 0 变为 `1`。
2.  **设置地址：** 将目标内部寄存器地址（`0x11`）写入地址 `0x31000` 处的 `req_addr` CSR。
3.  **发起读取：** 将 `READ` 操作码（`1`）写入地址 `0x31008` 处的 `req_op` CSR。
4.  **轮询响应：** 读取地址 `0x31014` 处的 `status` CSR，等待 bit 1 变为 `1`。
5.  **检查状态：** 读取地址 `0x31010` 处的 `rsp_op` CSR，确认操作成功。
6.  **读取数据：** 从地址 `0x3100c` 处的 `rsp_data` CSR 读取结果。
7.  **应答响应：** 写入地址 `0x31014` 处的 `status` CSR 以清除该响应。

## AXI CSR 接口

这些寄存器被映射到 CoralNPU 的 CSR 地址空间中，用于与调试模块通信。

| 地址    | 名称       | 描述                                                                                                |
|------------|------------|------------------------------------------------------------------------------------------------------------|
| 0x30800    | req_addr   | 在此写入目标调试模块寄存器地址。                                                       |
| 0x30804    | req_data   | 在此写入调试模块操作所需的数据。                                                            |
| 0x30808    | req_op     | 向该寄存器写入操作类型（例如 READ、WRITE）以发起一条调试模块命令。          |
| 0x3080c    | rsp_data   | 命令完成后，数据结果在此处可用。                                              |
| 0x30810    | rsp_op     | 命令完成后，状态结果（例如 SUCCESS、FAILED）在此处可用。                    |
| 0x30814    | status     | 一个只读寄存器，用于检查调试模块的状态。Bit 0 表示模块是否已准备好接收新请求。Bit 1 表示是否有响应可用。 |

## 调试模块内部寄存器

调试模块实现了一组内部寄存器，外部调试器可通过 AXI CSR 接口访问它们。这些寄存器用于控制和监视核。

| 地址 | 名称         | 描述                               |
|---------|--------------|-------------------------------------------|
| 0x04    | data0        | 抽象命令（abstract command）的数据寄存器。      |
| 0x10    | dmcontrol    | 调试模块控制寄存器。            |
| 0x11    | dmstatus     | 调试模块状态寄存器。             |
| 0x12    | hartinfo     | Hart 信息寄存器。                |
| 0x16    | abstractcs   | 抽象命令状态寄存器。         |
| 0x17    | command      | 抽象命令寄存器。                |

### dmcontrol (0x10)

`dmcontrol` 寄存器用于控制调试模块和核。

| 位（Bits）  | 名称      | 描述                                                                 |
|-------|-----------|-----------------------------------------------------------------------------|
| 31    | haltreq   | 请求停机核。                                                   |
| 30    | resumereq | 请求恢复核。                                                 |
| 29:2  | reserved  | 保留供未来使用。                                                    |
| 1     | ndmreset  | 复位调试模块。                                                     |
| 0     | dmactive  | 激活调试模块。                                                  |

### dmstatus (0x11)

`dmstatus` 寄存器提供调试模块和核的状态。

| 位（Bits）  | 名称       | 描述                                                                 |
|-------|------------|-----------------------------------------------------------------------------|
| 11    | allrunning | 所有 hart 都在运行。                                                      |
| 10    | anyrunning | 任意 hart 在运行。                                                        |
| 9     | allhalted  | 所有 hart 都已停机。                                                       |
| 8     | anyhalted  | 任意 hart 已停机。                                                         |
| 7:4   | reserved   | 保留供未来使用。                                                    |
| 3:0   | version    | 调试模块版本。                                                       |

### abstractcs (0x16)

`abstractcs` 寄存器提供抽象命令的状态。

| 位（Bits）  | 名称     | 描述                                                                 |
|-------|----------|-----------------------------------------------------------------------------|
| 12    | busy     | 抽象命令当前正在执行。                                |
| 11    | reserved | 保留供未来使用。                                                    |
| 10:8  | cmderr   | 抽象命令错误码。                                                |
| 7:0   | reserved | 保留供未来使用。                                                    |

### command (0x17)

`command` 寄存器用于向核发出抽象命令。

| 位（Bits）    | 名称      | 描述                                                                 |
|---------|-----------|-----------------------------------------------------------------------------|
| 31:24   | cmdtype   | 抽象命令的类型。                                               |
| 23:0    | control   | 命令特定的控制信息。                                       |

## 抽象命令（Abstract Commands）

抽象命令用于在核上执行复杂操作，例如读写寄存器。

要发出一条抽象命令，调试器写入 `command` 寄存器。`cmdtype` 字段指定命令类型，`control` 字段包含命令特定的信息。

### 访问寄存器命令（Access Register Command，cmdtype=0）

访问寄存器命令用于读写核寄存器。

访问寄存器命令的 `control` 字段格式如下：

| 位（Bits）    | 名称      | 描述                                                                 |
|---------|-----------|-----------------------------------------------------------------------------|
| 16      | write     | 1 表示写，0 表示读。                                                |
| 15:0    | regno     | 要访问的寄存器编号。                                       |

寄存器编号定义如下：

*   `0x0000-0x0FFF`：CSR
*   `0x1000-0x101F`：标量通用寄存器（Scalar GPRs）
*   `0x1020-0x103F`：浮点通用寄存器（Floating-point GPRs）

## 命令示例

### 读取一个 GPR（a0）

1.  **发出读命令：** 向 `command` 寄存器（地址 `0x17`）写入以下值：
    *   `cmdtype`（bits 31:24）：`0x0`（访问寄存器）
    *   `write`（bit 16）：`0`（读）
    *   `regno`（bits 15:0）：`0x100A`（对应 `a0`）
2.  **等待完成：** 轮询 `abstractcs` 寄存器（地址 `0x16`），直到 `busy` 位（bit 12）被清除。
3.  **检查错误：** 再次读取 `abstractcs` 寄存器，确认 `cmderr` 字段（bits 10:8）为 `0`。
4.  **读取结果：** 如果没有错误，从 `data0` 寄存器（地址 `0x04`）读取 GPR `a0` 的值。

### 写入一个 GPR（a0）

1.  **写入值：** 将 GPR `a0` 期望的值写入 `data0` 寄存器（地址 `0x04`）。
2.  **发出写命令：** 向 `command` 寄存器（地址 `0x17`）写入以下值：
    *   `cmdtype`（bits 31:24）：`0x0`（访问寄存器）
    *   `write`（bit 16）：`1`（写）
    *   `regno`（bits 15:0）：`0x100A`（对应 `a0`）
3.  **等待完成：** 轮询 `abstractcs` 寄存器（地址 `0x16`），直到 `busy` 位（bit 12）被清除。
4.  **检查错误：** 再次读取 `abstractcs` 寄存器，确认 `cmderr` 字段（bits 10:8）为 `0`。
